# SCRM 私域运营功能补齐需求规划

## 一、背景

当前 SCRM 运营台已实现 19 个基础管理模块（渠道/客户/社群/活码/素材/自动化/Agent/知识库等），但缺少大量实际私域运营核心功能。本规划补齐内容营销、会员运营、裂变活动、企微特有、数据运营、交易转化、触达运营 7 大领域，以及 MCP 框架化与多 AI 客户端适配基础设施。

## 二、架构分层原则

**核心原则：框架层是"能力引擎"，业务层是"场景编排者"。**

### 框架层（multi_tenant_saas）— 只提供可复用能力引擎

框架层不关心业务场景，只提供原子能力：

| 能力 | 已有基础 | 本轮规划 |
|------|---------|---------|
| 优惠券引擎 | `CouponService` + `Coupon`/`CouponUsage` 已有 | 扩展营销场景 API（批量发券、裂变发券） |
| 积分账户 | `CreditAccount`/`CreditTransaction` 已有 | 扩展积分规则引擎 |
| 短信通道 | `SmsService` 已有（验证码） | 扩展营销短信（模板管理、批量发送） |
| 表单引擎 | 无 | **新增** `FormBuilderService`（通用表单构建+数据收集） |
| 抽奖引擎 | 无 | **新增** `LotteryService`（概率控制、防刷、奖品池） |
| 投票引擎 | 无 | **新增** `VotingService`（排行榜、防刷票） |
| MCP 基础设施 | 无 | **新增**（详见第六章） |

### SCRM 业务层（scrm-platform）— 场景编排 + 调用框架能力

| 领域 | 模块 | 归属 | 说明 |
|------|------|------|------|
| 内容营销 | 海报管理 | SCRM | 强依赖活码/渠道追踪 |
| | 朋友圈 SOP | SCRM | 企微特有场景 |
| | 话术库 | SCRM | 已有 `ScriptService`，强依赖对话场景 |
| 会员运营 | 会员等级 | SCRM | 升级条件依赖 SCRM 客户数据 |
| | 积分规则+商城 | SCRM | 调用框架层 `CreditAccount` |
| | 成就体系 | SCRM | 强依赖客户行为数据 |
| | 会员卡/储值卡 | SCRM | 调用框架层 `CreditAccount` |
| 裂变活动 | 优惠券营销 | SCRM | 调用框架层 `CouponService` |
| | 裂变活动 | SCRM | 已有 `ReferralService`，强依赖客户关系链 |
| | 抽奖活动 | SCRM | 调用框架层 `LotteryService` |
| | 问卷表单 | SCRM | 调用框架层 `FormBuilderService` |
| | 投票活动 | SCRM | 调用框架层 `VotingService` |
| 企微运营 | 群发助手 | SCRM | 强依赖客户标签/渠道 |
| | 客户欢迎语 | SCRM | 已有 `AutoWelcomeService` |
| | 离职继承 | SCRM | 已有 `CustomerInheritanceService` |
| 数据分析 | RFM 深度画像 | SCRM | 强依赖 SCRM 客户/消费数据 |
| | 转化漏斗 | SCRM | 强依赖 CustomerJourney/StageTransition |
| | 流失预警 | SCRM | 强依赖客户互动/会话数据 |
| | 社群活跃度 | SCRM | 强依赖 Community/CommunityMember |
| 触达运营 | 定时触达 | SCRM | 已有 `ScheduledMarketingService` |
| | 短信营销 | SCRM | 调用框架层 `SmsService` |
| 交易转化 | 商品管理 | SCRM | 轻量商品，非完整电商 SKU |
| | 分销管理 | SCRM | 强依赖 SCRM 客户关系链 |

### 架构关系图

```
multi_tenant_saas（框架层 — 能力引擎）
├── CouponService/Coupon        ← 已有，优惠券核销引擎
├── CreditAccount/Transaction   ← 已有，积分账户
├── SmsService                  ← 已有，短信通道
├── FormBuilderService          ← 新增，表单引擎
├── LotteryService              ← 新增，抽奖引擎
├── VotingService               ← 新增，投票引擎
└── McpServer（新增）            ← MCP 协议 + 多客户端管理

scrm-platform（业务层 — 场景编排）
├── 内容营销: 海报/朋友圈SOP/话术库
├── 会员运营: 会员等级/积分规则+商城/成就/会员卡+储值卡
│   └── → 调用框架层 CreditAccount
├── 裂变活动: 优惠券营销/裂变活动/抽奖/问卷/投票
│   └── → 调用框架层 Coupon/Lottery/FormBuilder/Voting
├── 企微运营: 群发助手/欢迎语/离职继承
├── 数据分析: RFM画像/转化漏斗/流失预警/社群活跃度
├── 交易转化: 商品管理/分销管理
└── 触达运营: 定时触达/短信营销
    └── → 调用框架层 SmsService
```

## 三、功能清单

### P0 — 内容营销（私域内容基建）

#### 3.1 海报管理 `PosterManagement.vue`

**后端**：`PosterController` + `PosterService` + `Poster` 模型

**功能**：
- 海报模板库（行业模板、节日模板、品牌模板）
- 动态海报生成（替换文字/图片/二维码参数）
- 参数二维码海报（渠道活码 + UTM 追踪）
- 海报分组管理
- 海报数据统计（扫码量、分享量、转化量）
- 海报审核流程（合规检查）

**API**：
```
GET    /v1/scrm/posters              # 海报列表
POST   /v1/scrm/posters              # 创建海报
GET    /v1/scrm/posters/{id}         # 海报详情
PUT    /v1/scrm/posters/{id}         # 更新海报
DELETE /v1/scrm/posters/{id}         # 删除海报
POST   /v1/scrm/posters/{id}/render  # 渲染海报（传入参数生成最终图片）
GET    /v1/scrm/posters/{id}/stats   # 海报数据统计
GET    /v1/scrm/posters/templates    # 模板列表
```

#### 3.3 话术库 / 快捷回复 `ScriptLibrary.vue`

**后端**：`ScriptController` + `ScriptService` + `Script` 模型

**功能**：

- 快捷回复配置（按员工/按渠道/按场景）
- 话术效果统计（使用次数、转化率）

**API**：
```

POST   /v1/scrm/scripts              # 创建话术
PUT    /v1/scrm/scripts/{id}         # 更新话术

POST   /v1/scrm/scripts/{id}/use     # 记录使用
GET    /v1/scrm/scripts/recommend    # AI 推荐话术（传入对话上下文）
```

## 六、MCP 框架化建设与多 AI 客户端适配

### 6.1 现状分析

当前 SCRM 层 MCP 实现（`McpController` + `McpToolRegistry`）存在三个问题：

1. **协议与业务耦合**：JSON-RPC 2.0 协议处理（`initialize`/`tools/list`/`tools/call`）和 SCRM 工具定义混在一起，每新增一个业务域都要重写整个 McpController
2. **单客户端硬编码**：仅支持 WorkBuddy，无法适配 Hermers、OpenClaw 等不同 AI 客户端
3. **Skill 文件手动维护**：当前 `scrm-ops.md` 是手动创建的，新增工具后需要手动同步

### 6.2 框架层 MCP 基础设施

```
multi_tenant_saas/src/Mcp/
├── McpServerController.php       # JSON-RPC 2.0 协议处理 + SSE 流式响应
├── McpToolRegistry.php           # 抽象基类：工具注册框架
├── McpException.php              # JSON-RPC 标准错误码（-32700/-32600/-32601/-32603）
├── McpClientRegistry.php         # 多 AI 客户端注册管理
├── McpSkillGenerator.php         # 自动生成客户端 Skill 文件/配置
├── McpMiddleware.php             # Sanctum Token 认证 + 租户隔离
├── McpRoute.php                  # Route::mcp() 宏，一行注册 MCP 端点
└── Contracts/
    ├── McpToolRegistryContract.php  # 工具注册接口
    └── McpClientContract.php        # 客户端接口
```

#### 6.2.1 McpServerController（框架层协议处理器）

**职责**：处理 JSON-RPC 2.0 请求，不关心具体工具实现

```
POST /api/v1/mcp

请求体（JSON-RPC 2.0）：
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "tools/call",
  "params": {
    "name": "search_customers",
    "arguments": {"keyword": "张三"}
  }
}

支持的 method：
- initialize          → 返回协议版本、能力声明
- tools/list          → 列出当前客户端可用的工具
- tools/call          → 调用指定工具（支持 SSE 流式响应）
- notifications/initialized  → 客户端就绪通知
```

**认证**：通过 `McpMiddleware` 从请求 Header 中提取 `Authorization: Bearer <token>`，验证 Sanctum Token 并解析租户上下文。

**SSE 流式**：当工具返回 `StreamResponse` 时，自动切换为 SSE 响应（`text/event-stream`），支持 Agent 对话流式输出。

#### 6.2.2 McpToolRegistry（抽象基类）

**职责**：定义工具注册契约，业务层只需继承并实现 `registerTools()`

```php
abstract class McpToolRegistry
{
    protected array $tools = [];

    abstract protected function registerTools(): void;

    public function listTools(): array  // 通用实现
    public function callTool(string $name, array $arguments): mixed  // 通用实现
    protected function tool(string $name, string $description, array $schema, callable $handler): void
}
```

**业务层使用示例**：

```php
class ScrmMcpToolRegistry extends McpToolRegistry
{
    protected function registerTools(): void
    {
        $this->tool('search_customers', '搜索客户', [...], fn($args) => $this->customerService->search($args));
        $this->tool('create_poster', '创建海报', [...], fn($args) => $this->posterService->create($args));
        $this->tool('create_coupon', '创建优惠券', [...], fn($args) => $this->couponService->create($args));
        // 23 个新模块的工具也在此注册
    }
}
```

#### 6.2.3 McpClientRegistry（多客户端管理）

**职责**：管理不同 AI 客户端的注册、工具白名单、Skill 文件生成

| 客户端 | 类型 | 配置格式 | 工具可见范围 | 典型场景 |
|--------|------|---------|-------------|---------|
| WorkBuddy | 桌面 AI 助手 | `.md` Skill 文件 | 全部 SCRM 工具 | 运营人员自然语言操作 SCRM |
| Hermers | AI 编码助手 | JSON 配置 | 仅开发/运维工具 | 开发人员调试 SCRM 接口 |
| OpenClaw | AI 安全审计 | JSON 配置 | 仅审计/合规工具 | 安全审计员检查合规 |

**客户端配置示例**：

```php
// config/mcp.php (框架层)
return [
    'clients' => [
        'workbuddy' => [
            'name' => 'WorkBuddy',
            'skill_format' => 'markdown',
            'tool_scope' => '*',  // 所有业务域工具
        ],
        'hermers' => [
            'name' => 'Hermers',
            'skill_format' => 'json',
            'tool_scope' => ['scrm.developer', 'scrm.admin'],
        ],
        'openclaw' => [
            'name' => 'OpenClaw',
            'skill_format' => 'json',
            'tool_scope' => ['scrm.audit', 'scrm.compliance'],
        ],
    ],
];
```

#### 6.2.4 McpSkillGenerator（自动生成 Skill 文件）

**职责**：根据客户端类型和工具注册表，自动生成对应格式的 Skill 文件/配置

```
GET /api/v1/mcp/workbuddy/skill  → 返回 Markdown 格式的 WorkBuddy Skill
GET /api/v1/mcp/hermers/config   → 返回 JSON 格式的 Hermers 配置
GET /api/v1/mcp/openclaw/config  → 返回 JSON 格式的 OpenClaw 配置
```

**WorkBuddy Skill 文件自动生成逻辑**：
1. 遍历 MCP 工具注册表中该客户端可见的工具
2. 按工具分组（客户查询/活动管理/数据分析...）
3. 生成自然语言使用示例（"搜索标签为 VIP 的客户" → `search_customers`）
4. 输出完整的 `.md` Skill 文件

**Hermers/OpenClaw 配置自动生成逻辑**：
1. 输出 MCP Server 连接信息（URL、认证方式）
2. 输出工具列表（name + description + inputSchema）
3. 输出 JSON 格式的客户端配置文件

#### 6.2.5 Route::mcp() 宏

**职责**：一行代码注册 MCP 端点

```php
// scrm-platform/routes/api.php
Route::mcp('scrm', ScrmMcpToolRegistry::class);
// 自动注册:
//   POST /api/v1/mcp              → McpServerController

//   GET  /api/v1/mcp/{client}/config  → McpSkillGenerator
//   GET  /api/v1/mcp/clients        → McpClientRegistry
```
